from tensorflow.contrib.layers import flatten

def LeNet(x, dropout):    
    
    #Y Reshape from 2D to 4D. This prepares the data for
    #Y convolutional and pooling layers.
    ##YH already in correct shape? (37248, 32, 32, 3)
    #     x = tf.reshape(x, (-1, 32, 32, 3))
    
    
    # Arguments used for tf.truncated_normal, randomly defines variables for the weights and biases for each layer
    mu = 0
    sigma = 0.1
    
    # TODO: Layer 1: Convolutional. Input = 32x32x3. Output = 28x28x6.
    # set weights
    wc1 = tf.Variable(tf.truncated_normal([5, 5, 3, 6], mu, sigma, name='wc1'))
    strides = 1
    
    # process convolution
    x = tf.nn.conv2d(x, wc1, strides=[1, strides, strides, 1], padding='VALID')
    
    # add bias
    bc1 = tf.Variable(tf.truncated_normal([6], mu, sigma, name='bc1'))
    x = tf.nn.bias_add(x, bc1)

    # TODO: Activation. relu = rectificed linear unit
    x = tf.nn.relu(x)

    # TODO: Pooling. Input = 28x28x6. Output = 14x14x6.
    k = 2
    x = tf.nn.max_pool(
        x,
        ksize=[1, k, k, 1],
        strides=[1, k, k, 1],
        padding='VALID')

    # TODO: Layer 2: Convolutional. Output = 10x10x16.
    wc2 = tf.Variable(tf.truncated_normal([5, 5, 6, 16], mu, sigma, name='wc2'))
    strides = 1
    x = tf.nn.conv2d(x, wc2, strides=[1, strides, strides, 1], padding='VALID')
    
    bc2 = tf.Variable(tf.truncated_normal([16], mu, sigma, name='bc2'))
    x = tf.nn.bias_add(x, bc2)
    
    # TODO: Activation.
    x = tf.nn.relu(x)

    # TODO: Pooling. Input = 10x10x16. Output = 5x5x16.
    k = 2
    x = tf.nn.max_pool(
        x,
        ksize=[1, k, k, 1],
        strides=[1, k, k, 1],
        padding='VALID')

    # TODO: Flatten. Input = 5x5x16. Output = 400.
    
    # reshape x as 2d matrix, tf.reshape with shape param with axis -1 infers size
    # so reshape with shape = [-1,400] reshapes input tensor to rows with col size = 400
    x = tf.reshape(x, [-1, 400])
    
    # TODO: Layer 3: Fully Connected. Input = 400. Output = 120.
    wd1 = tf.Variable(tf.truncated_normal([5*5*16, 120], mu, sigma, name='wd1'))
    bd1 = tf.Variable(tf.truncated_normal([120], mu, sigma, name='bd1'))
    x = tf.add(tf.matmul(x, wd1), bd1)

    # TODO: Activation.
    x = tf.nn.relu(x)
    
    #YH TODO: try dropout?
    x = tf.nn.dropout(x, dropout)
    
    # TODO: Layer 4: Fully Connected. Input = 120. Output = 84.
    wd2 = tf.Variable(tf.truncated_normal([120, 84], mu, sigma, name='wd2'))
    bd2 = tf.Variable(tf.truncated_normal([84], mu, sigma, name='bd2'))
    x = tf.add(tf.matmul(x, wd2), bd2)
    
    # TODO: Activation.
    x = tf.nn.relu(x)
    x = tf.nn.dropout(x, dropout)

    # TODO: Layer 5: Fully Connected. Input = 84. Output = 43.
    # 43 classes
    classes = 43
    wd3 = tf.Variable(tf.truncated_normal([84, classes], mu, sigma, name='wd3'))
    bd3 = tf.Variable(tf.truncated_normal([classes], mu, sigma, name='bd3'))
    x = tf.add(tf.matmul(x, wd3), bd3)
    
    return x

Architecture Pipeline:¶

  • Input 32x32x3: i.e. 3 channel color images as input
  • Convolution Layer patch size 5x5x3 with 6 filters and 'VALID' (no) padding producing 28x28x6 outputs -- convolution layers are used to detect features at each level, going from fine to coarse.
  • activation layer using Relu (rectified linear unit) which is the simplest non-linear activation function. It should also be suited for a large input ranges (versus 0 to 1 for sigmoid activation or -1 to 1 for tanh activation functions)
  • max pool 2x2 kernel, 2x2 stride: max pooling is usually used to reduce overfitting, but we may not want it here
  • conv 5x5 patch size with 6 filters, reducing outputs down to 14x14x6, the second convolution layer is used to produce higher level features.
  • activation: another Relu
  • max pooling, again 2x2 kernel, 2x2 stride, may not want this here either
  • flatten to 400 nodes in prep for fully connected layers to follow
  • fully connected layer from 400 nodes to 120
  • activation, do another Relu
  • fully connected layer 120 -> 84 nodes
  • activation: Relu
  • fully connected layer 84 -> 43 to the number of classes

Question 5¶

What approach did you take in coming up with a solution to this problem? It may have been a process of trial and error, in which case, outline the steps you took to get to the final solution and why you chose those steps. Perhaps your solution involved an already well known implementation or architecture. In this case, discuss why you think this is suitable for the current problem.

Answer: The initial approach is to use the LeNet architecture since it showed good results for the MNIST data set.

The LeNet architecture uses convolution layers at the beginning of the pipeline. Convolutional layers are used to detect features that are statistically invariant. In the case of images, features at different scales are location space invariant across the image dimenstions. Low level features are aggregated into higher level features for each subsequent layer. In this traffic sign classification problem, the initial layer may detect lines and edges at different angles. The next layer may consolidate the primitive geometric features into circles, squares, etc.

The activation function used is ReLU (rectified linear unit) as it is both the simplest non-linear function (as compared to sigmoid and tanh for example) and well suited to domain values not restricted to limit range (e.g. 0 to 1 and -1 to -1 for tanh)

Max pooling is used after convolution layers to reduce overfitting and reduce learning parameters dimensionality.

Drop out was tried for the fully connected layers. Usually drop out is used for regularization to reduce overfitting. Drop out values (keep probability) of 0.75 and 0.5 were tried. It did not improve the performance of the system, and slowed learning improvement considerably.

Learning rates of 0.001 and 0.0001 were tried. Although learning rate of 0.0001 did produce better validation and test accuracy, it did not produce a system that performed better for the new images tried.
